# WordPress / WooCommerce Integration

**Status:** ACTIVE

## Connection Details

- **Site:** https://stolengoat.com
- **Credentials:** `wordpress-credentials.json` (project root, in .gitignore)
- **WP REST API auth:** HTTP Basic Auth (username + application password)
- **WC REST API auth:** Query string (consumer_key + consumer_secret)

## URL Format Quirk

LiteSpeed blocks `/wp-json/` pretty permalinks for WordPress core endpoints. The two APIs use different URL patterns:

| API | URL Format | Auth Method |
|---|---|---|
| WP REST API | `{site}/?rest_route=/wp/v2/pages&...` | HTTP Basic Auth |
| WC REST API | `{site}/wp-json/wc/v3/...` | Query string (`consumer_key`, `consumer_secret`) |

**Do not mix these up.** WP endpoints will 403 if you use `/wp-json/wp/v2/...`.

## How to Use

Both wrappers are in `tools/wordpress/`. Import and call directly from Python scripts.

### WooCommerce (categories, products)

```python
import sys
sys.path.insert(0, 'C:/ClaudeProjects/pablo')
from tools.wordpress.woocommerce import get_category, get_categories, find_category, update_category, get_product

# Fetch a category by ID
cat = get_category(64)
print(cat['name'], cat['slug'], cat['description'])

# Find a category by slug
cat = find_category('socks')
print(cat['id'], cat['name'])

# Get all categories
all_cats = get_categories()
print(f"{len(all_cats)} categories")

# Update a category (name changes both title tag and H1)
update_category(64, name="Men's Cycling Jerseys", description="<p>New description here.</p>")

# Fetch a product
product = get_product(12345)
print(product['name'], product['price'])
```

### WordPress (pages)

```python
import sys
sys.path.insert(0, 'C:/ClaudeProjects/pablo')
from tools.wordpress.wordpress import get_page, get_pages, find_page, update_page

# Fetch a page by ID
page = get_page(2)
print(page['title']['rendered'])

# Find a page by slug
page = find_page('about')
print(page['id'], page['title']['rendered'])

# List pages
pages = get_pages(per_page=20)
for p in pages:
	print(p['id'], p['slug'], p['title']['rendered'])

# Update a page
update_page(2, title="New Title", content="<p>New content</p>")
```

## SEO Context — No Plugin Installed

There is **no SEO plugin** on stolengoat.com (no Yoast, Rank Math, or AIOSEO). This means:

- **Title tags** are auto-generated: `{category_name} – Stolen Goat`
- **H1** on category pages = category name
- **Title and H1 cannot be set independently** — both are derived from the WooCommerce category `name` field
- **Meta descriptions** cannot be set via the API — there is no mechanism to store them without an SEO plugin
- The only SEO-relevant fields we can change via API are `name` (affects title + H1) and `description` (on-page content below the product grid)

### What Can Be Changed

| Element | API Field | Effect |
|---|---|---|
| Title tag | `name` | Becomes `{name} – Stolen Goat` |
| H1 | `name` | Displayed as the page heading |
| On-page description | `description` | Category description content (HTML) |
| Slug | `slug` | URL path segment |

### What Cannot Be Changed (without an SEO plugin)

| Element | Issue |
|---|---|
| Custom title tag | Would need Yoast/Rank Math to set independently of name |
| Meta description | No storage mechanism without an SEO plugin |
| OG tags | Not customisable per-page |
| Canonical URL | Auto-generated by WordPress |

## Rules

1. **All changes must be approved by Tim first** — never push updates to production without explicit approval
2. **Read before write** — always fetch current state before making changes
3. **Test on a single category first** — when making bulk changes, update one and verify before proceeding
4. **Preserve existing descriptions** — when updating, append to or refine existing content rather than replacing unless directed otherwise
5. **Do not expose credentials** — never log, display, or commit credentials from `wordpress-credentials.json`
6. **Use `{'User-Agent': 'Pablo/1.0'}` header** — both wrappers include this automatically
